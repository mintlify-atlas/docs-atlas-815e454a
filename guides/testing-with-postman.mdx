This guide walks you through testing the SOLARE API using Postman, including authentication workflows, collection organization, and practical examples for all major endpoints.

## Prerequisites

Before you begin testing:

<Steps>
  <Step title="Install Postman">
    Download and install [Postman](https://www.postman.com/downloads/) for your platform
  </Step>
  
  <Step title="Verify API is Running">
    Ensure the Laravel server is running:
    ```bash
    php artisan serve
    ```
    The API should be accessible at `http://127.0.0.1:8000`
  </Step>
  
  <Step title="Set Base URL">
    All endpoints use the base URL: `http://127.0.0.1:8000/api`
  </Step>
</Steps>

<Note>
  Always include the header `Accept: application/json` in all requests to ensure proper JSON responses.
</Note>

## Setting Up Your Postman Collection

### Create a New Collection

1. Click **New Collection** in Postman
2. Name it "SOLARE API"
3. Add collection-level variables:

| Variable | Initial Value | Description |
|----------|--------------|-------------|
| `base_url` | `http://127.0.0.1:8000/api` | API base URL |
| `token` | (empty) | Will store auth token |
| `admin_email` | `admin@solare.com` | Admin test account |
| `admin_password` | `password123` | Admin password |
| `customer_email` | `cliente@solare.com` | Customer test account |
| `customer_password` | `password123` | Customer password |

### Configure Collection Authorization

1. Go to collection **Authorization** tab
2. Type: **Bearer Token**
3. Token: `{{token}}`

This allows all requests in the collection to automatically use the stored token.

## Authentication Workflow

### 1. Customer Registration

Create a new customer account:

<CodeGroup>
```json Request
POST {{base_url}}/register
Content-Type: application/json
Accept: application/json

{
  "nombre": "Juan",
  "apellido_paterno": "Pérez",
  "apellido_materno": "González",
  "correo": "juan.perez@example.com",
  "contrasena": "SecurePass123",
  "contrasena_confirmation": "SecurePass123",
  "telefono": "+52 55 1234 5678"
}
```

```json Response (201 Created)
{
  "message": "Usuario registrado exitosamente como Cliente",
  "access_token": "eyJ0eXAiOiJKV1QiLCJhbGc...",
  "token_type": "Bearer",
  "user": {
    "id": 15,
    "nombre": "Juan",
    "apellido_paterno": "Pérez",
    "correo": "juan.perez@example.com",
    "rol": {
      "id": 3,
      "nombre": "Cliente"
    }
  }
}
```
</CodeGroup>

<Accordion title="Auto-save Token Script">
  Add this to the **Tests** tab of your login/register requests to automatically save the token:
  
  ```javascript
  if (pm.response.code === 201 || pm.response.code === 200) {
      const response = pm.response.json();
      if (response.access_token) {
          pm.collectionVariables.set("token", response.access_token);
          console.log("Token saved:", response.access_token.substring(0, 20) + "...");
      }
  }
  ```
</Accordion>

### 2. Login

Authenticate existing users:

<CodeGroup>
```json Customer Login
POST {{base_url}}/login
Content-Type: application/json
Accept: application/json

{
  "correo": "{{customer_email}}",
  "contrasena": "{{customer_password}}"
}
```

```json Admin Login
POST {{base_url}}/login
Content-Type: application/json
Accept: application/json

{
  "correo": "{{admin_email}}",
  "contrasena": "{{admin_password}}"
}
```

```json Success Response (200 OK)
{
  "message": "Inicio de sesión exitoso",
  "access_token": "eyJ0eXAiOiJKV1QiLCJhbGc...",
  "token_type": "Bearer",
  "user": {
    "id": 1,
    "nombre": "Admin",
    "apellido_paterno": "SOLARE",
    "correo": "admin@solare.com",
    "rol": "Administrador"
  }
}
```
</CodeGroup>

### 3. Get User Profile

Verify authentication:

```json
GET {{base_url}}/profile
Authorization: Bearer {{token}}
Accept: application/json
```

### 4. Logout

Revoke the current token:

```json
POST {{base_url}}/logout
Authorization: Bearer {{token}}
Accept: application/json

# Response (200 OK)
{
  "message": "Sesión cerrada exitosamente"
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="401 Unauthorized">
    **Cause:** Missing or invalid authentication token
    
    **Solutions:**
    - Ensure you've logged in and the token is saved
    - Check that Authorization header is set: `Bearer {{token}}`
    - Token may have expired - login again
    - Verify the `Accept: application/json` header is present
  </Accordion>
  
  <Accordion title="403 Forbidden">
    **Cause:** User doesn't have required role permissions
    
    **Solutions:**
    - Check the endpoint's required roles in the [User Roles guide](/guides/user-roles)
    - Login with an account that has appropriate permissions
    - For admin endpoints, use `admin@solare.com`
    - For customer endpoints, use `cliente@solare.com`
  </Accordion>
  
  <Accordion title="422 Validation Error">
    **Cause:** Request body doesn't meet validation requirements
    
    **Solutions:**
    - Check the error response for specific field issues
    - Ensure required fields are included
    - Verify email format and uniqueness
    - Password must be minimum 8 characters
    - `contrasena_confirmation` must match `contrasena`
  </Accordion>
  
  <Accordion title="404 Not Found">
    **Cause:** Resource doesn't exist or wrong endpoint
    
    **Solutions:**
    - Verify the endpoint URL is correct
    - Check that the resource ID exists (product, variant, etc.)
    - Ensure base URL includes `/api`
  </Accordion>
</AccordionGroup>
