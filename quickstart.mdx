Get started with the SOLARE API by making your first requests. This guide will walk you through authentication, browsing products, and creating an order.

<Note>
  **Base URL**: `http://127.0.0.1:8000/api`
  
  All requests must include the header: `Accept: application/json`
</Note>

## Prerequisites

Before you begin, ensure you have:
- The SOLARE API server running locally (`php artisan serve`)
- A tool to make HTTP requests (Postman, cURL, or similar)
- Test credentials (provided below)

<Steps>
  <Step title="Authenticate and Get Your Token">
    Start by logging in to obtain a Bearer token. You'll need this token for all authenticated requests.

    <CodeGroup>
    ```bash cURL
    curl -X POST http://127.0.0.1:8000/api/login \
      -H "Accept: application/json" \
      -H "Content-Type: application/json" \
      -d '{
        "correo": "cliente@solare.com",
        "contrasena": "password123"
      }'
    ```

    ```javascript JavaScript
    const response = await fetch('http://127.0.0.1:8000/api/login', {
      method: 'POST',
      headers: {
        'Accept': 'application/json',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        correo: 'cliente@solare.com',
        contrasena: 'password123'
      })
    });

    const data = await response.json();
    console.log(data.access_token);
    ```

    ```python Python
    import requests

    response = requests.post(
        'http://127.0.0.1:8000/api/login',
        headers={'Accept': 'application/json'},
        json={
            'correo': 'cliente@solare.com',
            'contrasena': 'password123'
        }
    )

    data = response.json()
    token = data['access_token']
    ```
    </CodeGroup>

    **Response** (200 OK):
    ```json
    {
      "message": "Inicio de sesión exitoso",
      "access_token": "eyJ0eXAiOiJKV1QiLCJhbGc...",
      "token_type": "Bearer",
      "user": {
        "id": 2,
        "nombre": "Cliente",
        "apellido_paterno": "Prueba",
        "correo": "cliente@solare.com",
        "rol": "Cliente"
      }
    }
    ```

    <Note>
      Save the `access_token` value. You'll use it in the `Authorization` header for subsequent requests.
    </Note>
  </Step>

  <Step title="Browse the Product Catalog">
    Explore available products without authentication. The catalog endpoint returns all active products with their variants, materials, and images.

    <CodeGroup>
    ```bash cURL
    curl -X GET http://127.0.0.1:8000/api/productos \
      -H "Accept: application/json"
    ```

    ```javascript JavaScript
    const response = await fetch('http://127.0.0.1:8000/api/productos', {
      headers: {
        'Accept': 'application/json'
      }
    });

    const products = await response.json();
    ```

    ```python Python
    response = requests.get(
        'http://127.0.0.1:8000/api/productos',
        headers={'Accept': 'application/json'}
    )

    products = response.json()
    ```
    </CodeGroup>

  <Step title="Get Product Details">
    Retrieve detailed information about a specific product by its ID.

    ```bash cURL
    curl -X GET http://127.0.0.1:8000/api/productos/1 \
      -H "Accept: application/json"
    ```

    This returns the same structure as the catalog, but for a single product with all its variants and related data.
  </Step>

  <Step title="Create Your First Order">
    Now let's place an order using your authentication token. You'll need the `variante_id` from the products you want to order.

    <CodeGroup>
    ```bash cURL
    curl -X POST http://127.0.0.1:8000/api/pedidos \
      -H "Accept: application/json" \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
      -d '{
        "productos": [
          {
            "variante_id": 1,
            "cantidad": 2
          },
          {
            "variante_id": 3,
            "cantidad": 1
          }
        ],
        "direccion_id": null,
        "notas": "Entrega en horario de oficina"
      }'
    ```

    ```javascript JavaScript
    const response = await fetch('http://127.0.0.1:8000/api/pedidos', {
      method: 'POST',
      headers: {
        'Accept': 'application/json',
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${accessToken}`
      },
      body: JSON.stringify({
        productos: [
          { variante_id: 1, cantidad: 2 },
          { variante_id: 3, cantidad: 1 }
        ],
        direccion_id: null,
        notas: 'Entrega en horario de oficina'
      })
    });

    const order = await response.json();
    ```

    ```python Python
    response = requests.post(
        'http://127.0.0.1:8000/api/pedidos',
        headers={
            'Accept': 'application/json',
            'Authorization': f'Bearer {token}'
        },
        json={
            'productos': [
                {'variante_id': 1, 'cantidad': 2},
                {'variante_id': 3, 'cantidad': 1}
            ],
            'direccion_id': None,
            'notas': 'Entrega en horario de oficina'
        }
    )

    order = response.json()
    ```
    </CodeGroup>

    **Response** (201 Created):
    ```json
    {
      "status": "success",
      "message": "Pedido creado exitosamente",
      "pedido_id": 1
    }
    ```

    <Warning>
      The order creation automatically deducts stock from inventory via database triggers. Ensure the variants have sufficient stock before placing orders.
    </Warning>
  </Step>

## Common Issues

<AccordionGroup>
  <Accordion title="401 Unauthorized - Token is invalid or expired">
    Your authentication token may have expired or is malformed. Request a new token by logging in again using the `/api/login` endpoint.
  </Accordion>

  <Accordion title="403 Forbidden - User does not have the required role">
    The endpoint you're trying to access requires a specific role. For example, only customers can create orders, and only administrators can create employee accounts.
  </Accordion>

  <Accordion title="422 Unprocessable Entity - Validation errors">
    Your request body is missing required fields or contains invalid data. Check the response message for specific validation errors.
  </Accordion>

  <Accordion title="500 Internal Server Error - Database or server issue">
    Check that:
    - The database is running and accessible
    - The `.env` file is properly configured
    - All migrations and seeders have been executed
  </Accordion>
</AccordionGroup>
