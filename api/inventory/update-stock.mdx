## Endpoint

```
PUT /api/inventario/{id}
```

## Description

This endpoint allows authorized staff to manually adjust the stock level of a specific product variant. All stock changes are tracked in the `MovimientoInventario` table with details including the type of movement, quantity changed, previous and new stock levels, and the user who made the change.

The endpoint supports three types of inventory movements:
- **entrada**: Add stock (increment)
- **salida**: Remove stock (decrement)
- **ajuste**: Set stock to an absolute value

All operations are wrapped in a database transaction to ensure data consistency.

## Authentication

This endpoint requires authentication via Laravel Sanctum token.

**Required Roles:** Warehouse management only
- Administrador
- Gerente
- Almacenista

**Header:**
```
Authorization: Bearer {token}
```

## Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the product variant (`variante_producto`) to update
</ParamField>

## Request Body

<ParamField body="tipo" type="string" required>
  Type of inventory movement. Must be one of:
  - `entrada`: Add stock (increases existencias by cantidad)
  - `salida`: Remove stock (decreases existencias by cantidad)
  - `ajuste`: Set stock to absolute value (sets existencias to cantidad)
  
  **Validation:** Required, must be exactly one of: `entrada`, `salida`, `ajuste`
</ParamField>

<ParamField body="cantidad" type="integer" required>
  The quantity to add, remove, or set (depending on tipo).
  
  - For `entrada`: Amount to add to current stock
  - For `salida`: Amount to subtract from current stock
  - For `ajuste`: New total stock level (absolute value)
  
  **Validation:** Required, must be an integer, minimum value of 1
  
  **Important:** Stock cannot go below zero. If a `salida` or `ajuste` would result in negative stock, the request will fail with an error.
</ParamField>

<ParamField body="motivo" type="string" optional>
  Optional description or reason for the stock change. If not provided, defaults to "Actualización manual de almacén".
  
  **Validation:** Optional, must be a string if provided
</ParamField>

## Business Logic

### Movement Type Behavior

1. **entrada (Entry)**: 
   - Adds `cantidad` to current stock
   - Formula: `nuevo_stock = existencias_actual + cantidad`
   - Creates a movement record with the specified cantidad

2. **salida (Exit)**:
   - Subtracts `cantidad` from current stock
   - Formula: `nuevo_stock = existencias_actual - cantidad`
   - Fails if result would be negative
   - Creates a movement record with the specified cantidad

3. **ajuste (Adjustment)**:
   - Sets stock to the exact `cantidad` value
   - Formula: `nuevo_stock = cantidad`
   - Records the difference as movement: `|nuevo_stock - existencias_actual|`
   - Fails if cantidad is less than 0

### Inventory Movement Tracking

Every successful stock update creates a record in `MovimientoInventario` with:
- `variante_id`: The variant being updated
- `tipo`: The movement type (entrada/salida/ajuste)
- `cantidad`: Amount changed (absolute difference for ajuste)
- `cantidad_anterior`: Stock level before update
- `cantidad_nueva`: Stock level after update
- `usuario_id`: ID of the authenticated user making the change
- `motivo`: Reason for the change (or default message)
- `fecha_movimiento`: Timestamp of the movement

### Transaction Safety

The entire operation runs in a database transaction, ensuring that:
- Either both the variant update AND movement record succeed
- Or both operations are rolled back if any error occurs
- Data consistency is maintained at all times
