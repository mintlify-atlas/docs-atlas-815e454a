---
title: 'Logout'
api: 'POST /api/logout'
---

Revokes the current user's access token, effectively logging them out. This endpoint requires authentication.

<Warning>
  This endpoint requires authentication. Include a valid Bearer token in the Authorization header.
</Warning>

## Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication. Format: `Bearer {access_token}`
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message confirming the session has been closed and the token revoked.
</ResponseField>

## Example Request

```bash
curl -X POST http://127.0.0.1:8000/api/logout \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJSUzI1NiJ9..."
```

## Example Response

```json
{
  "message": "Sesión cerrada exitosamente"
}
```

## Error Responses

### Unauthenticated (401)

```json
{
  "message": "Unauthenticated."
}
```

This error occurs when:
- No Authorization header is provided
- The token is invalid or malformed
- The token has already been revoked
- The token has expired

<Note>
  After successful logout, the revoked token cannot be used for subsequent API requests. The user must login again to obtain a new token.
</Note>
