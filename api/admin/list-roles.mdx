## Overview

This endpoint returns all role records from the system's `roles` table. It is primarily used to populate role selection dropdowns when creating employee accounts or managing user permissions.

The roles define the permission levels and access control within the SOLARE system:
- **Administrador** - Full system access
- **Gerente** - Management-level access to reports and inventory
- **Cliente** - Customer access for ordering products
- **Supervisor** - Supervisory access to reports and monitoring
- **Vendedor** - Sales staff access to orders and catalog
- **Almacenista** - Warehouse staff access to inventory management

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication. User must have the **Administrador** role.
</ParamField>

<Warning>
  This endpoint is restricted to administrators only. It returns sensitive role information used for access control configuration.
</Warning>

## Response

Returns an array of all role objects.

    <ResponseField name="nombre" type="string">

    <ResponseField name="actualizado_en" type="string">

    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

### 401 Unauthorized

```json
{
  "message": "Unauthenticated."
}
```

Returned when:
- No Authorization header is provided
- The Bearer token is invalid or expired

### 403 Forbidden

```json
{
  "message": "No tiene permisos para acceder a este recurso."
}
```

Returned when the authenticated user does not have the Administrador role.

## Implementation Notes

- Returns all records from the `roles` table without filtering
- No pagination is applied (assumes a small, fixed number of roles)
- Results are returned in the order they exist in the database
- Controller location: `app/Http/Controllers/Auth/AuthController.php:72`
- Route defined at: `routes/api.php:30`
- Protected by both `auth:sanctum` and `role:Administrador` middleware

## Use Cases

### Employee Creation Form

Use this endpoint to populate a role selection dropdown when creating employee accounts:

```javascript
fetch('http://127.0.0.1:8000/api/admin/roles', {
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Accept': 'application/json'
  }
})
.then(response => response.json())
.then(roles => {
  // Exclude Cliente role from employee creation
  const employeeRoles = roles.filter(role => role.nombre !== 'Cliente');
  
  // Populate select dropdown
  const select = document.getElementById('role-select');
  employeeRoles.forEach(role => {
    const option = document.createElement('option');
    option.value = role.id;
    option.textContent = role.nombre;
    select.appendChild(option);
  });
});
```

### Role-Based Access Control

Use the role IDs to configure permission checks in your application:

```javascript
const ROLES = {
  ADMIN: 1,
  MANAGER: 2,
  CUSTOMER: 3,
  SUPERVISOR: 4,
  SALESPERSON: 5,
  WAREHOUSE: 6
};

function canAccessReports(userRoleId) {
  return [ROLES.ADMIN, ROLES.MANAGER, ROLES.SUPERVISOR].includes(userRoleId);
}

function canManageInventory(userRoleId) {
  return [ROLES.ADMIN, ROLES.MANAGER, ROLES.WAREHOUSE].includes(userRoleId);
}
```

## Database Schema

The `roles` table structure:

```sql
CREATE TABLE roles (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  nombre VARCHAR(50) NOT NULL UNIQUE,
  descripcion TEXT NULL,
  creado_en TIMESTAMP NULL,
  actualizado_en TIMESTAMP NULL
);
```
